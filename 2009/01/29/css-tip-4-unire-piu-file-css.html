CSS tip #4 - Unire più file .css

Se siete amanti dei codici ben organizzati o se vi capita di lavorare sullo stesso file css con più persone, questo può essere uno di quei "tips&trick" di cui è meglio essere a conoscenza. Quando si aggiunge un file .css all'interno di un documento HTML facciamo un'operazione di importazione, quello che non tutti sanno è che anche i file .css a loro volta possono importare altri file (sempre .css) con il vantaggio di mantenere organizzato in modo migliore la gestione dei fogli di stile, sopratutto per eventuali modifiche future. Per importare altri file .css all'interno di uno "principale" basta usare questa semplice sintassi: 
...
@import url("nome_file.css");
...
In CSS è facile superare le centinaia di righe e nonostante i commenti siano un valido aiuto, a volte dover cercare alcune regole specifiche magari solo per una pagina, può non essere cosi immediato. Immaginando un sito con diverse pagine, dove due di queste utilizzano degli stili particolari, un buon metodo per poter sfruttare "@import" è quello di scrivere le regole specifiche di quelle pagine in due file .css a parte e importarli in quello che contiene gli stili globali. Nel caso invece più persone stiano lavorando alla stessa pagina, ognuna di queste può utilizzare un proprio file .css e importarlo in uno globale, cosi che eventuali modifiche vengano "sovrapposte" ma non cancellate, ed evitando anche che ognuno abbia aperto una diversa "versione" del file .css su cui si sta lavorando.
